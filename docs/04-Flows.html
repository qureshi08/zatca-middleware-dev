1. Customer Journey — onboarding, end-to-end FR-AUTH / CONN / ONB / ENV

From zero to compliant. One-time. Customer onboards in Demo mode (ZATCA simulation, OTP 123456) — clearly flagged — then deliberately switches to Real to go live (see Flow 7).

2. ZATCA Onboarding — detail FR-ONB-1..6 · FR-ENV

CSR → Compliance CSID → compliance checks → Production CSID. Done in Demo (ZATCA simulation) first; switch to Real later (see Flow 7).

⭐ 3. Core Automatic Compliance Flow — async + queued FR-FLOW-1..10 · FR-WB · FR-ERR

The heart of the product. No manual middleware step — staff just create an invoice in their own software.

4. Invoice Classification — decision logic FR-FLOW-2 · FR-DOC

How a document is routed to clearance vs reporting.

5. Failure & Retry FR-FLOW-10 · FR-ERR-1..5

Transient failures retry automatically; permanent ones notify the customer with the reason.

6. Headless API Flow — Mode B FR-API

Custom systems drive the same compliance engine.

7. Environment Promotion — Demo → Real FR-ENV-1..7

Controlled move from Demo (ZATCA simulation, OTP 123456) to Real (live core).

Legend & Conventions

  • Demo = ZATCA simulation endpoint (OTP 123456, not legally filed). Real = ZATCA core (real OTP, legally filed). See 08-Environments-and-Release-Management.
  • Pull mode (recommended): the middleware fetches the full document. Push mode: the ERP sends the full payload.
  • ⭐ Flow 3 is the canonical path the whole product exists to deliver.
  • All flows are per tenant and isolated.
